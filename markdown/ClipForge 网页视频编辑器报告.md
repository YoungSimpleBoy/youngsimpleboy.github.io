# ClipForge 网页视频编辑器报告

## 1. 工具概述

ClipForge 是一个运行在浏览器端的视频编辑工具，主要用于视频片段截取与多格式导出。
本工具部署在静态站点中，无需后端服务，用户可直接在网页上传本地视频进行处理。

- 工具入口：toolkit/video_editor/index.html
- 目标场景：快速切片、轻量导出、社交媒体动图生成
- 支持格式：MP4、WebM、GIF

## 2. 功能清单

### 2.1 视频导入与预览

- 支持本地视频文件上传。
- 提供播放器预览、当前播放时间显示。

### 2.2 切片区间设置

- 支持起始与结束秒数输入。
- 支持双滑块调节区间。
- 支持“以当前播放点设为开始/结束”快速定位。

### 2.3 导出参数控制

- 导出格式可选：MP4 / WebM / GIF。
- 可选导出宽度：保持原始或常见分辨率。
- 视频质量可选：高质量 / 平衡 / 小体积。
- 支持是否保留音频（视频格式下）。
- GIF 可单独设置帧率。

### 2.4 导出结果管理

- 导出完成后提供结果预览。
- 提供一键下载导出文件。
- 显示导出文件名和体积。

## 3. 技术实现说明

### 3.1 前端技术栈

- HTML + CSS + JavaScript 原生实现。
- 视频处理核心：ffmpeg.wasm（浏览器端 FFmpeg）。

### 3.2 编码策略

- MP4：H.264（libx264）+ AAC（可选）。
- WebM：VP9（libvpx-vp9）+ Opus（可选）。
- GIF：palettegen + paletteuse 两阶段流程，兼顾色彩与体积。

### 3.3 目录结构

- 页面结构：toolkit/video_editor/index.html
- 样式文件：toolkit/video_editor/styles.css
- 逻辑脚本：toolkit/video_editor/app.js
- 本地依赖：toolkit/video_editor/vendor/ffmpeg

## 4. 问题与修复记录

### 4.1 现象

首次加载 ffmpeg.wasm 时出现 Worker 构造失败，提示跨域脚本不可访问。

### 4.2 原因

ffmpeg 的 UMD worker chunk 由第三方域名提供时，某些本地开发域名下会触发 Worker 访问限制。

### 4.3 修复方案

- 将 ffmpeg.js 与对应 worker chunk（814.ffmpeg.js）改为同源本地加载。
- 保留 ffmpeg core 的双 CDN 回退策略（jsDelivr 主源，unpkg 备用），提升初始化稳定性。

### 4.4 结果

引擎可正常加载，切片与导出功能恢复可用。

## 5. 使用流程

1. 打开工具页面并选择本地视频。
2. 设置开始和结束时间。
3. 选择导出格式与参数。
4. 加载引擎并执行导出。
5. 预览结果并下载文件。

## 6. 已知限制

- 浏览器端转码对内存和 CPU 占用较高，长视频或高分辨率处理时间较长。
- GIF 文件在长时长场景下体积增长明显，建议缩短时长或降低宽度与帧率。

## 7. 后续优化建议

1. 增加导出进度条与预计剩余时间。
2. 增加预设方案（短视频、动图、高清存档）。
3. 增加批量切片与多片段导出能力。
4. 增加关键帧快速定位与时间轴标记。

## 8. 结论

ClipForge 已满足“网页端视频切片 + 多格式导出”的核心需求，并在跨域 Worker 问题修复后具备稳定可用性，适合个人站点与轻量创作场景。
